# 認証・ユーザー管理マイクロサービス テスト手順書

この手順書は、認証サービス（Authentication Service）とユーザー管理サービス（User Management Service）の機能テストを行うためのガイドです。

## 📋 このドキュメントの目的

- **単体テスト**: 各サービスが個別に正しく動作することを確認
- **統合テスト**: 両サービスが連携して正しく動作することを確認  
- **障害テスト**: 一方のサービスが停止した場合の動作を確認
- **Azure Service Bus連携**: イベント駆動アーキテクチャの動作確認

各テストケースには、実行手順と期待される結果を明記しているので、開発・QA担当者が迷わずテストを実行できます。

## 🔧 テスト環境の準備

### 💻 必要なソフトウェア

テストを実行する前に、以下のソフトウェアをインストールしてください：

- **Java 21** - アプリケーション実行用
- **Maven** - ビルドツール
- **Docker & Docker Compose** - データベース環境構築用
- **PostgreSQL** - メインデータベース
- **Redis** - イベント伝播用（ローカル環境）

### 🚀 環境構築の手順

**ステップ1: リポジトリのクローン**

```bash
git clone https://github.com/your-organization/skishop.git
cd skishop
```

**ステップ2: Docker環境の起動**

```bash
docker-compose up -d
```

💡 この操作で以下のサービスが起動します：

- PostgreSQL（認証サービス用）
- PostgreSQL（ユーザー管理サービス用）  
- Redis（イベント伝播用）

**ステップ3: データベースの初期化**

```bash
# 認証サービス用データベースを初期化
cd authentication-service
psql -h localhost -U postgres -d auth_db -f init-auth-db.sql

# ユーザー管理サービス用データベースを初期化
cd ../user-management-service
psql -h localhost -U postgres -d user_db -f init-db.sql
```

**ステップ4: アプリケーションの起動**

それぞれ別のターミナルで起動してください：

```bash
# ターミナル1: 認証サービス
cd authentication-service
./mvnw spring-boot:run -Dspring.profiles.active=local

# ターミナル2: ユーザー管理サービス
cd user-management-service
./mvnw spring-boot:run -Dspring.profiles.active=local
```

**ステップ5: テストスクリプトの準備**

```bash
# テスト用スクリプトに実行権限を付与
chmod +x user-management-service/test-api.sh
chmod +x authentication-service/test-api.sh
```

### ✅ 環境の動作確認

以下のコマンドでサービスが正常に起動しているか確認します：

```bash
# 認証サービスの動作確認
curl http://localhost:8080/actuator/health
# 期待する結果: {"status":"UP"}

# ユーザー管理サービスの動作確認  
curl http://localhost:8081/actuator/health
# 期待する結果: {"status":"UP"}
```

🎉 両方のサービスが `{"status":"UP"}` を返せば、テスト環境の準備完了です！

## 📝 テスト実行時の注意点

1. **環境確認**: テスト実行前に必ず両サービスが起動していることを確認してください
2. **データクリーンアップ**: テスト後はテストデータを適切にクリーンアップしてください
3. **ログ確認**: エラーが発生した場合は、両サービスのログを確認してください
4. **メトリクス監視**: 本番環境でのテスト時は、メトリクスやアラートを監視してください
